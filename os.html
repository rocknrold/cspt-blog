What is an Operating System?

An operating system (OS) in a basic definition is an intermediary interface/software that connects a computer user to the hardware.

Remember on the previous topic from computer hardware systems the layer on the middle that’s sits between the hardware layer and the application layer is the Operating system and known as the OS for short.

The above figure shows that the Operating System is in between the Software Layer where user interacts and the hardware layer where the OS controls and communicates to.

What Operating Systems Do?

Another way to look on OS is that, operating system is a piece of software that manages files, manages memory, manages processes, handles input and output, and controls peripheral devices like disk drives and printers, among other things.

A government is comparable to an operating system. That, like the nation, has little valuable purpose of its own. It essentially creates a setting in which other programs can perform useful tasks.

Operating-System Operations

Process Management

In a structure, a process is the unit of work. A system is made up of a number of processes, some of which are operating-system processes (those that run system code) and others which run user code. By multiplexing on a single CPU, for example, both of these processes may actually run at the same time. The operating system is responsible for the following activities in connection with process management:

  • Scheduling processes and threads on the CPUs
  • Creating and deleting both user and system processes
  • Suspending and resuming processes
  • Providing mechanisms for process synchronization
  • Providing mechanisms for process communication

Memory Management

General-purpose machines must retain many programs in memory to increase both CPU efficiency and the speed at which the machine responds to its users, necessitating memory management. There are several memory management systems in use. These schemes represent a variety of methods, and the success of each algorithm varies depending on the case. When it comes to memory control software, there are a few things to consider.

The operating system is responsible for the following activities in connection with memory management:

  • Keeping track of which parts of memory are currently being used and who is using them
  • Deciding which processes (or parts of processes) and data to move into and out of memory
  • Allocating and deallocating memory space as needed

Storage Management

The operating system offers a structured, logical vision of information storage to make the data system more user-friendly. To describe a logical storage unit, the disk, the operating system abstracts from the physical properties of its storage devices. Files are mapped into physical media by the operating system, which then accesses them through storage devices.

The operating system is responsible for the following activities in connection with file management:

  • Creating and deleting files
  • Creating and deleting directories to organize files
  • Supporting primitives for manipulating files and directories
  • Mapping files onto secondary storage
  • Backing up files on stable (nonvolatile) storage media

Protection and Security

If a computer system has multiple users and requires multiple processes to run at the same time, data access must be controlled. Mechanisms ensure that only certain processes that have received proper permission from the operating system can access directories, memory segments, CPU, and other resources.

Memory-addressing hardware, for example, means that a process can only run within its own address space. No mechanism will gain hold of the CPU without ultimately relinquishing it, thanks to the timer. The security of the different peripheral systems is preserved and device-control registers are not open to users.

Kernel

Many kernel-mode processes can be involved in the operating system at any given time. As a result, there are many potential race conditions in the code that implements an operating system (kernel code). Consider a kernel data structure that keeps track of all the available files in the system. When a new file is opened or closed, this list must be updated (by adding the file to the list).

The different changes to this list could trigger a race condition if two processes open files at the same time. Structures for managing memory allocation, process lists, and interrupt management are examples of kernel data structures that are vulnerable to race conditions. Kernel engineers are responsible for ensuring that the operating system is free of certain race conditions.

A network operating system provides an environment in which users, who

either logging in to the appropriate remote machine or transferring data from the remote machine to their own machines. Currently, all general-purpose operating systems, and even embedded operating systems such as Android and iOS, are network operating systems.

Embedded systems

Embedded computers are the most prevalent form of computers in existence. These devices are found everywhere, from car engines and manufacturing robots to DVDs and microwave ovens. They tend to have very specific tasks. The systems they run on are usually primitive, and so the operating systems provide limited features. Usually, they have little or no user interface, preferring to spend their time monitoring and managing hardware devices, such as automobile engines and robotic arms.

These embedded systems vary considerably. Some are general-purpose

special-purpose applications to implement the functionality. Others are hardware devices with a special-purpose embedded operating system providing just the functionality desired. Yet others are hardware devices with application specific integrated circuits (ASICs) that perform their tasks without an operating system.
