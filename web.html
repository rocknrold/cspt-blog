What is a Web Platforms?

Many people is confuse when talking about a platform some may think that all website may be a platform but it is not.

In this section, let’s dive into Web Platforms – identify what it is and gives some examples of technologies that it is used and explain this technology.

So what really a Web Platform is?

A web platform is a stack of different technologies or services that provides functionality for us to create or maintain our website or the application we’re building. It enables the development by providing different capabilities such as supports for different content management systems, continuous development, scalable and reliability for applications, this may help for faster and robust building, deployment, and management of powerful web applications.

Additional Explanation:

A web platform usually consists of the Web Application, Mobile Applications and the communication between this and other external services. In addition to a more complete integration between each of the elements that comprise it.

(Mackaber, 2018)

Platforms that provide services that are useful to websites and web-based software as a service such as web servers, web application servers, content delivery networks and edge computing.

(Spacey, 2021)

Web Platform technologies applied

As stated Web Platforms can also be used to provide services for web applications and is very useful to a websites and the different web service applications.

As an example is the Microsoft Azure Web App service

Azure App Service is an HTTP-based service for hosting web applications, REST APIs, and mobile back ends. You can develop in your favorite language, be it .NET, .NET Core, Java, Ruby, Node.js, PHP, or Python. Applications run and scale with ease on both Windows and Linux-based environments.

App Service not only adds the power of Microsoft Azure to your application, such as security, load balancing, autoscaling, and automated management. You can also take advantage of its DevOps capabilities, such as continuous deployment from Azure DevOps, GitHub, Docker Hub, and other sources, package management, staging environments, custom domain, and TLS/SSL certificates.

With App Service, you pay for the Azure compute resources you use. The compute resources you use are determined by the App Service plan that you run your apps on. 

Also a Web Platform is applied to a Web hosting platforms, providers such as GoDaddy are web hosting platforms that gives services for integrating different applications, maintaining and scale.

We may think of Web platforms as integrations of different services, example is the Nginx web server and in a web platform it is included or you can run this kind of service, host a content delivery network, or provide an API that can be access by users and is included in the application. Also integrate frameworks that are written in different programming languages.

Conclusion

In this section, I’ am going to wrap everything up. When choosing or using a web platforms, it should be clear that the purpose is for automating processes, adding functionality such as scaling the application, integrating services for application and maintain parts/services of applications.
